# Claude Code 活用ガイド - このプロジェクトでの使い方

## Claude Codeとは？

Claude Codeは、ターミナルから直接Claudeとコーディング作業ができるコマンドラインツールです。
通常のClaude.aiのチャットよりも、コーディングに特化した機能を持っています。

## このプロジェクトでClaude Codeを使うメリット

### 1. **ファイル操作が簡単**
- 複数のファイルを一度に作成・編集できる
- ディレクトリ構造を自動で作成
- コードの修正がスムーズ

### 2. **コンテキストの維持**
- プロジェクト全体の構造を理解している
- 前回の作業内容を覚えている
- 一貫性のあるコードを生成

### 3. **即座のフィードバック**
- コードを書いたらすぐに実行して確認
- エラーがあればその場で修正
- 動作確認がスピーディー

## Claude Codeのセットアップ

### インストール方法

画像に表示されていた「GitHubに接続」ボタンから始めます。

1. Claude.aiで「Claude Code」の通知をクリック
2. GitHubアカウントと連携
3. コマンドラインツールをインストール

### 基本的な使い方

```bash
# Claude Codeを起動
claude code

# 特定のプロジェクトで起動
claude code minimap_renderer_gui

# ファイルを指定して起動
claude code app.py
```

## このプロジェクトでの推奨ワークフロー

### 方法1: Claude.ai + 手動実装（現在の方法）
```
1. Claude.aiで設計・相談
2. 提示されたコードを手動でファイルに貼り付け
3. 動作確認
4. 問題があればClaude.aiに報告
```

**メリット**: 各ステップを理解しながら進められる
**デメリット**: コピペが多く、時間がかかる

### 方法2: Claude Code + 自動実装（推奨）
```
1. ターミナルでClaude Codeを起動
2. 「Phase 1.2の実装をお願いします」と指示
3. Claudeが自動でファイルを作成・編集
4. 動作確認
5. 問題があればその場で修正依頼
```

**メリット**: 高速、正確、手間が少ない
**デメリット**: 初回セットアップが必要

### 方法3: ハイブリッド（バランス型）
```
1. 設計・要件定義はClaude.aiで
2. 実装フェーズはClaude Codeで
3. レビューと確認はClaude.aiで
```

**メリット**: 両方の利点を活かせる
**デメリット**: ツールを切り替える必要がある

## このプロジェクトでの具体的な使用例

### 例1: Phase 1.2の実装

**Claude Codeでの指示例:**
```
minimap_renderer_guiプロジェクトで以下を実装してください:

1. DEVELOPMENT_GUIDE.mdのPhase 1.2に記載されている内容
2. app.py, templates/index.html, static/css/style.css, static/js/app.jsを作成
3. 作成後、python app.pyで起動して動作確認
```

Claude Codeが自動的に:
- 必要なディレクトリを作成
- すべてのファイルを作成
- コードの整合性を確認
- 起動コマンドを実行

### 例2: バグ修正

**Claude Codeでの指示例:**
```
app.pyでファイルアップロード時にエラーが出ています。
エラーメッセージ: [エラー内容をコピペ]

原因を調査して修正してください。
```

Claude Codeが:
- エラーの原因を特定
- コードを修正
- 修正内容を説明
- 再テストを提案

### 例3: 機能追加

**Claude Codeでの指示例:**
```
進捗表示をアニメーション付きにしたいです。
style.cssとapp.jsを修正して、より視覚的に分かりやすくしてください。
```

Claude Codeが:
- 関連ファイルを特定
- コードを更新
- 変更内容を説明
- プレビュー方法を提示

## Claude Codeを使った開発の流れ（推奨）

```bash

```

```
「Phase 1.2に進んでください」
```

## よくある質問

### Q: Claude.aiとClaude Codeのどちらを使うべき？
A: 以下の使い分けがおすすめです:
- **相談・設計・要件定義**: Claude.ai（会話がしやすい）
- **実装・デバッグ・修正**: Claude Code（ファイル操作が速い）
- **レビュー・ドキュメント作成**: Claude.ai（読みやすい）

### Q: Claude Codeは日本語で使える？
A: はい、日本語での指示が可能です。ただし、エラーメッセージなどは英語の場合があります。

### Q: 前回の作業内容は引き継がれる？
A: Claude Codeは同じプロジェクトディレクトリ内であれば、ファイルの内容から状況を把握できます。
ただし、PROJECT_SPEC.mdやDEVELOPMENT_GUIDE.mdを参照させると、よりスムーズです。

### Q: 複数の会話セッションにまたがっても大丈夫？
A: 大丈夫です。以下の情報を伝えれば、すぐに続きから開始できます:
```
「minimap_renderer_guiプロジェクトの開発を続けます。
DEVELOPMENT_GUIDE.mdを参照してください。
現在Phase 1.2まで完了しています。Phase 1.3を実装してください。」
```

## このプロジェクトでClaude Codeを試すための最初のステップ

### 準備
1. Claude.aiの通知から「GitHubに接続」をクリック
2. Claude Codeをインストール
3. ターミナル（PowerShellまたはコマンドプロンプト）を開く

### 初回実行
```bash
# プロジェクトディレクトリに移動
cd %USERPROFILE%\Documents\minimap_renderer_gui

# Claude Codeを起動
claude code

# 以下のように指示
「このプロジェクトはWoWs Minimap RendererのGUI化プロジェクトです。
DEVELOPMENT_GUIDE.mdに従って、Phase 1.1から実装を開始してください。」
```

### 期待される結果
- Claudeがディレクトリ構造を作成
- 必要なファイルを生成
- .gitignoreを作成
- git commitまで実行

## まとめ

このプロジェクトでは:
1. **設計・計画**: Claude.ai（このチャット）
2. **実装**: Claude Code（試してみたい場合）
3. **相談・レビュー**: Claude.ai

という流れが効率的です。Claude Codeはオプションですが、試してみる価値は十分にあります！
